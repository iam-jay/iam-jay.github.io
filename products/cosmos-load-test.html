CosmosLoadTest

A .NET 8 command-line tool that drives a configurable load against Azure Cosmos DB (NoSQL) across all nine document operations — Read, Query, Create, ReadFeed, Upsert, Patch, Replace, Delete and transactional Batch. You set the request rate, total operations, document size and the percentage mix per operation; it paces the workload, tracks every request, and produces a clean HTML report with throughput, latency percentiles and requested-vs-actual operation mix.

View source on GitHub ↗ Deploy to Azure ↗

🚀 One click spins up a Windows VM that downloads, builds and runs the workload — fill the form in the Azure Portal, no setup required.

Features

Built for realistic load

  • ✓All 9 document operations. Read, Query, Create, ReadFeed, Upsert, Patch, Replace, Delete and transactional Batch.
  • ✓Configurable mix. Set a percentage per operation; any operation at 0% is skipped.
  • ✓Quota-based scheduler. Paces to a global requests/sec; when an operation hits its target it drops out and its share redistributes.
  • ✓Smart document pool. Pre-seeds data when Create isn't in the mix; otherwise reuses documents created during the run.
  • ✓HTML report. Totals, success/error, ops-per-minute, latency p50/p95/p99, average RU and requested-vs-actual mix.
  • ✓One-click Azure deploy. An ARM template + portal wizard spin up a Windows VM that builds and runs it automatically.
